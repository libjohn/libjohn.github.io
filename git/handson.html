Git Workshop - Hands-On

Generate SSH Keys

This must be done on each workstation you use to push.

  • https://gitlab.oit.duke.edu/help/ssh/README

This step was covered in the Outline Section

Make a Repository

  1. Duke’s GitLab – https://gitlab.oit.duke.edu
  2. Duke Shibboleth Login

  3. New Project

  4. Add a project name. Use the name test-one

    • --> test-one
    • – Create Project
    • On the Next Page, notice the section below Command line instructions. That information could come in handy. For future repositories, you may want to make a note of those Git commands.

  5. Make a README file


    1. Open in a New Tab, click the README link
    2. Commit changes

  6. Make a license file (Open in a New Tab, click the LICENSE link)


    1. click Appply a license format > choose MIT (or paste in a Creative Commons License)
    2. Commit changes

  7. Make a .gitignore file (Open in a New Tab, click the .gitignore link)

    1. click Apply a .gitignore format

      • choose R

    2. Commit changes

  8. Congratulations. You’ve just created your first GitLab repository. Click test-one in the top-left of your screen and you’ll see your repository – including the three files you just created.

Clone

Using RStudio, clone your repository. This will pull down the whole repo including the three files you just created.

Bring the repo down locally via RStudio

  1. Copy GitLab URL to clipboard

    • – Copy the repo URL
    • repo https://gitlab.oit.duke.edu/<<your-NetID>>/test-one, copyURL to clipboard (click icon)

  2. Launch RStudio and create a New Project…

    1. Menubar: File > New Project… > Version Control > Git > paste the URL from the step above into Repository URL:

    2. Create Project

Push: Edit and Commit

This time you’ll make local changes by editing your local repository. After saving the changes you’ll add and commit the changes. Lastly, you will push your changes up to the remote repoistory.

Edit

  1. In the Files Pane, Open the README.md file

  2. In the Editor Pane, modify your README with Markdown

    • Make a Subheading with the word “Purpose”: ##Purpose
    • A blank line, then “This is my first Git repository. I’m going to push this file via RStudio”

  3. Save the changes to this file

Commit & Push

  1. In the Git pane:

    1. Check each of the staged files
    2. Click commit and provide a message: this is my first edit of a README and supporting files
    3. Click the commit button to engage your commit process with the message, then close
    4. Click the Push arrow, then close, then close the commit dialogue box

  2. View your changes back at GitLab: https://gitlab.oit.duke.edu/<<your-NetID>>/test-one

Pull

The next time you open your project in RStudio the first step you should take is to Pull from the GitLab repo. Then make your changes, then commit and push. If you forget to Pull you make have to resolve some conflicts so get into the habit of pulling before you make edits.
